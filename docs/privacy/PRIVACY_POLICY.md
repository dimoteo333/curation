# 큐레이터 개인정보 처리방침

최종 수정일: 2026-04-17

## 1. 수집하는 데이터

큐레이터는 사용자가 직접 가져온 파일의 내용만 처리합니다.

- 사용자가 선택한 `.txt`, `.md` 파일의 제목과 본문
- 파일 형식, 수정 시각, 가져온 시각 같은 최소한의 로컬 메타데이터
- 온디바이스 검색을 위한 임베딩 벡터

큐레이터는 연락처, 위치, 광고 식별자, 서버 계정 정보 같은 데이터를 수집하지 않습니다.

## 2. 데이터 저장 위치

가져온 개인 기록은 기기 내부의 암호화된 SQLite 데이터베이스에 저장됩니다.

- 제목, 본문, 태그, 메타데이터는 암호화되어 저장됩니다.
- 암호화 키는 기기별로 처음 실행할 때 생성되며, iOS Keychain 또는 Android Keystore 기반 secure storage에 보관됩니다.
- 임베딩 벡터는 개인 식별 정보가 아니므로 검색 성능을 위해 암호화하지 않을 수 있습니다.

## 3. 기기 밖으로 나가는 데이터

온디바이스 모드에서는 개인 기록과 질문 내용이 기본적으로 기기 밖으로 전송되지 않습니다.

- 로컬 검색과 로컬 생성 경로를 우선 사용합니다.
- 원격 점검 모드를 사용하지 않는 한, 개인 기록 파일은 서버로 업로드되지 않습니다.

## 4. 보관 기간

가져온 데이터는 사용자가 삭제할 때까지 기기에 남아 있습니다.

- 사용자가 새로운 파일을 가져오면 해당 데이터가 추가되거나 같은 ID의 로컬 레코드가 갱신될 수 있습니다.
- 사용자가 앱 설정에서 "모든 데이터 삭제"를 실행하면 로컬 DB와 앱 설정이 제거됩니다.

## 5. 사용자의 삭제 권리

사용자는 언제든지 앱 설정에서 "모든 데이터 삭제"를 실행하여 저장된 로컬 데이터를 삭제할 수 있습니다.

- 이 작업은 로컬 SQLite 파일과 관련 인덱스를 제거합니다.
- 앱 설정과 온보딩 상태를 포함한 `SharedPreferences` 데이터도 함께 삭제됩니다.
- 삭제 후에는 복구할 수 없습니다.

## 6. 문의 및 정책 변경

본 저장소에서 정책이 변경되면 이 문서를 함께 업데이트합니다.
