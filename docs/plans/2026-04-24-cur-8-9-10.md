# CUR-8 CUR-9 CUR-10 Plan

## Scope

- Mobile calendar import source selection and persisted opt-out state
- Mobile local-data reset behavior in `SharedPreferences`
- README and privacy/architecture documentation accuracy

## Constraints

- Preserve FastAPI HTTP contract compatibility.
- Keep calendar import additive to the current mobile-only ingest flow.
- Preserve user settings during destructive local-data deletion.

## Implementation

1. Add persisted calendar source exclusions backed by `SharedPreferences` and filter calendar imports against that opt-out list.
2. Expose available calendar sources in settings as a simple checkbox list and keep the stored exclusions reactive in the UI.
3. Replace `SharedPreferences.clear()` in `deleteAllData()` with explicit removal of data-only keys while preserving settings such as onboarding, runtime mode, calendar sync preference, model paths, and excluded calendar sources.
4. Update delete/reset copy, privacy docs, and architecture docs so they describe preserved settings correctly.
5. Rewrite `README.md` so shipped functionality is separated from roadmap-only functionality with explicit planned status markers.

## Risks

- Calendar listing depends on platform permission state, so the UI must degrade cleanly when permission is missing or restricted.
- Preserving onboarding/runtime settings changes recovery-reset behavior; tests and user-facing copy must be updated together.
- Old installs may not yet have the new excluded-calendar preference key, so defaults must remain inclusive.
