# Visual Direction

## Brand baseline

- 핵심 인상은 `조용한 회상`, `따뜻한 기록`, `정리된 질문`
- 밝은 바탕 위 gold-cream 계열을 메인으로 사용
- 앱 아이콘과 홈 로고의 둥근 사각형, 원형 탐색 mark를 web hero에서도 반복 사용

## Color

Primary:
- warm gold
- pale cream

Support:
- paper beige
- ink brown
- muted sage for reassurance accents

Usage:
- hero gradient는 gold -> cream -> paper 흐름
- 기능 설명 카드에는 pure white 대신 살짝 paper tint 사용
- 진한 검정보다 brown ink를 사용해 제품 톤을 맞춘다

## Type

- headline: 존재감 있는 serif 또는 humanist display
- body: 읽기 쉬운 sans
- web과 App Store 모두 "기록"과 "회상"의 결을 살리는 방향 유지

## Layout

- landing hero는 왼쪽 copy, 오른쪽 dual-device mockup
- 아래 섹션은 `문제 -> 질문 -> 답변 -> 원문 -> 관리` 순서
- App Store screenshot은 한 장에 한 메시지만 남긴다

## Motion

- landing page 초기 진입에서 mockup 두 장이 20~32px 정도 시차를 두고 부드럽게 올라오게 한다
- card hover는 shadow 강화보다 tint 변화 위주로 간다
- App Store artwork는 정지 이미지 기준으로 설계하고, web만 최소한의 motion을 쓴다

## Asset rules

- 제품 캡쳐는 `captures/ios-simulator/core/*.png`를 원본으로 사용한다
- mockup 편집 시 실제 UI 색을 과하게 바꾸지 않는다
- logo mark는 흰색/cream 단색 사용을 우선한다
- 환경 파일이나 외부 SaaS 링크는 이 디렉터리에 두지 않는다

## Copy tone

- 진단/처방처럼 들리는 표현은 피한다
- "당신의 기록", "다시 읽기", "오늘의 질문"처럼 조용하고 직접적인 문장을 우선한다
- 과장보다 신뢰와 구체성을 앞세운다
