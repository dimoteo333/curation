# Curation Landing Page Brief

## Goal

- "내 기록만으로 답을 찾는 온디바이스 큐레이션"을 5초 안에 이해시킨다.
- iOS 앱 설치 의사결정을 만든다.
- 개인정보/로컬 처리 신뢰를 첫 화면에서 확보한다.

## Primary audience

- 일기, 메모, 캘린더를 꾸준히 남기지만 다시 읽지는 않는 사람
- 감정 패턴, 회복 루틴, 반복된 질문을 자기 기록 안에서 찾고 싶은 사람
- 외부 업로드 없는 개인용 AI 해석 도구를 찾는 사람

## Core message

헤드라인:
- 당신의 기록만으로, 오늘의 답을 찾습니다

서브카피:
- 큐레이터는 일기, 메모, 캘린더를 기기 안에서 다시 읽고 지금의 질문에 맞는 맥락을 조용히 연결합니다.

Primary CTA:
- App Store에서 만나보기

Secondary CTA:
- 화면 미리보기 보기

## Page structure

### 1. Hero

사용 asset:
- `captures/ios-simulator/core/home.png`
- `captures/ios-simulator/core/answer.png`

구성:
- 좌측: headline, subcopy, CTA 2개
- 우측: home -> answer stacked device mockup
- 신뢰 배지 3개: `온디바이스 우선`, `개인 기록 기반`, `파일/캘린더 지원`

### 2. Why now

문제 제기:
- 기록은 쌓이는데, 정작 필요한 순간엔 찾지 못한다.
- 검색은 단어를 찾고, 큐레이션은 맥락을 찾는다.

짧은 카드 3개:
- 흩어진 하루를 다시 연결
- 반복되는 감정 패턴 탐색
- 회복 루틴과 관계 맥락 회수

### 3. Product flow

사용 asset:
- `captures/ios-simulator/core/today-question.png`
- `captures/ios-simulator/core/answer.png`
- `captures/ios-simulator/core/memory-sheet.png`

3-step narrative:
1. 질문한다
2. 기록을 연결한다
3. 원문까지 다시 본다

### 4. Timeline and memory depth

사용 asset:
- `captures/ios-simulator/core/timeline.png`
- `captures/ios-simulator/core/settings.png`

핵심 문장:
- 큐레이터는 한 번의 답변으로 끝나지 않고, 타임라인과 설정에서 기록의 축적을 계속 관리하게 만든다.

포인트:
- 타임라인 기반 회상
- 파일/캘린더 입력 관리
- 로컬 데이터 상태 가시성

### 5. Privacy and trust

카피:
- 당신의 질문과 기록은 기본적으로 기기 안에서 먼저 처리됩니다.
- 데이터를 더 넣을수록 더 개인적인 답을 돌려주지만, 그 출발점은 항상 당신의 로컬 기록입니다.

보조 항목:
- 외부 전송 없이 시작 가능
- 사용자가 직접 불러온 기록만 사용
- 설정에서 데이터 상태 확인 가능

### 6. Final CTA

헤드라인:
- 지금의 질문을, 당신의 기록으로 다시 읽어보세요

CTA:
- App Store에서 다운로드

## Visual notes

- 배경은 따뜻한 paper tone과 옅은 grain 사용
- 제품색은 앱 아이콘의 gold-cream 계열을 기준으로 통일
- mockup 배치는 단일 정면 1장보다 약간 겹친 2장 구성이 더 적합
- 너무 기술문서처럼 보이지 않도록 copy block은 짧게 유지

- `기록 기반`
- `답변 + 근거 기록 + 원문 회수`
- `일기 / 메모 / 캘린더`
